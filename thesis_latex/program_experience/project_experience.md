## 项目一：高性能前向折叠计算框架的性能优化与加速

**项目角色**：核心开发者 / 算法优化负责人

**技术栈**：Python, PyTorch, NumPy, Linux Profiling, 稀疏矩阵(CSR), 分块计算, 函数级缓存(LRU Cache)

**项目描述**：

**Situation**：在大规模粒子物理实验的数据分析中，需要对海量模拟样本（百万量级）执行前向折叠能谱拟合。核心计算涉及高维矩阵运算（560×5600 响应矩阵构建、5600×5600 微分截面映射矩阵乘法）以及大量特殊函数（误差函数 erf）的密集调用。单次拟合耗时约 110 秒，总计算规模达 500 万次拟合（约 2×10⁵ CPU·h），在常规算力资源下难以为继。

**Task**：设计并实现一套基于 PyTorch 的高性能计算框架，在不牺牲数值精度（相对误差 < 10⁻⁶）的前提下，系统性地降低单次拟合耗时，使百万级拟合任务在有限集群资源上可行。

**Action**：
- **性能剖析与瓶颈定位**：通过 Profiling 分析单次拟合的时间分布，定位到信号谱计算占总耗时的 98.6%，其中响应矩阵构造占 66.7%、微分截面映射占 30.8%，从而确定优化优先级。
- **分层缓存策略**：将拟合流程中的中间计算量分为静态量（预计算并常驻内存）和动态量（基于 `functools.lru_cache` 实现函数级 memoization），消除重复计算，获得约 **3× 加速**。
- **带状稀疏矩阵表示**：分析微分截面映射矩阵的物理结构，发现其非零元素集中于主对角带附近；采用 PyTorch 稀疏存储格式（CSR），将矩阵-向量乘法复杂度从 O(N²) 降至 O(N·w)（w 为带宽），获得约 **1.5× 加速**，同时显著降低内存占用。
- **分块张量化构造**：将 560×5600 响应矩阵沿列方向切分为固定宽度（512）的子块，逐块计算并累加。该策略充分利用 CPU 缓存局部性（data locality），减少内存搬运开销，响应矩阵构造时间从 55ms 降至 27ms，贡献约 **1.5× 加速**。

**Result**：
- 三类优化叠加后，单次拟合耗时从 **110.1s 降至 15.2s**，实现总体约 **7× 加速**；其中信号谱计算从 108.6s 降至 14.3s（约 8× 加速）。
- 数值精度以双精度计算为基准，相对误差控制在 **10⁻⁶ 量级**以内。
- 该框架成功支撑了 500 万次拟合的全量分析任务，相关成果已发表于《European Physical Journal C》（2025），并被国际实验合作组采纳为基础分析工具，具有良好的通用性和可迁移性。

## 项目二：大规模数值模拟驱动的数据分箱策略优化

**项目角色**：核心开发者 / 数据策略研究负责人

**技术栈**：Python, NumPy/SciPy, 大规模 Monte Carlo 模拟, χ² 统计推断, 协方差矩阵分析, 参数化降维

**项目描述**：

**Situation**：在大型探测器实验的数据分析中，观测能谱的分箱（binning）策略直接影响参数估计的偏差与精度。分箱过细会导致边缘箱统计量稀疏，使高斯近似失效并引入系统性偏差；分箱过粗则丢失谱形信息，降低灵敏度。此外，输入模型能谱的分段策略需要在"模型灵活性"与"外部数据约束能力"之间找到最优平衡，这是一个典型的高维参数优化问题。

**Task**：通过大规模数值模拟，定量评估不同分箱/分段策略对关键参数测量精度的影响，给出可操作的最优策略建议，并验证其在实际分析中的稳健性。

**Action**：
- **大规模 Toy-MC 仿真框架**：搭建支持 5,000~10,000 个伪实验的自动化仿真流水线，每组实验覆盖多种分箱配置（从 56 bins 到 560 bins），实现参数扫描与结果统计的全自动化。
- **统计偏差的定量评估**：基于分位数方法（非参数化）定义偏差指标，通过伪实验分布的第 16/84 百分位点估计置信区间，避免对分布正态性的假设依赖。
- **分段参数化与降维优化**：将连续能谱的无限维泛函空间离散化为有限维的段归一化因子 {αᵢ}，实现"段内刚性、段间自由"的参数化策略。通过联合 χ² 目标函数与协方差矩阵约束，将高维优化问题转化为可控的有限维参数估计。
- **固定-扰动对比实验**：设计两组对照实验——固定模型（评估统计行为）与扰动模型（评估模型依赖性），在多种分段宽度下比较参数精度，系统量化模型依赖引入的额外系统误差。

**Result**：
- 确定重建能谱最优分箱宽度：将 bin 宽从 20keV 调整至 100keV 后，关键参数的统计偏差从 **0.8σ 降至 < 0.2σ**，有效抑制了低统计量下的高斯近似失效问题。
- 确定模型能谱最优分段宽度：在约 **300 keV** 段宽处，模型依赖引入的系统误差达到最小（**< 0.3%**），实现了模型灵活性与统计约束的最优折中。
- 研究成果已被国际实验合作组采纳，直接应用于首篇物理结果论文的振荡分析，为实验的分箱策略提供了定量依据。

## 项目三：高并发统计推断引擎与分布式覆盖率验证系统

**项目角色**：核心开发者 / 统计推断方案负责人

**技术栈**：Python, PyTorch, 分布式计算, HPC 集群调度, 大规模并行拟合, Monte Carlo 方法, 似然比检验

**项目描述**：

**Situation**：在精确参数测量中，传统的渐近统计方法（Wilks 定理）在低统计量或似然面高度非线性的场景下会失效——实际的检验统计量分布显著偏离卡方分布，导致置信区间的覆盖率不正确。具体表现为：似然面存在多个局部极小值（multi-modal），轮廓曲线呈现强烈的非二次型特征，单次点估计可能严重偏离真值。这要求采用 Feldman-Cousins（FC）方法通过海量伪实验构造经验分布，但其计算规模极为庞大。

**Task**：设计并实现一套高并发的统计推断引擎，在 HPC 集群上完成百万级拟合任务，构造关键参数的经验检验统计量分布，验证置信区间的覆盖率，并为实验合作组提供可靠的统计策略建议。

**Action**：
- **计算规模分析与任务编排**：将 FC 构造分解为 50 个参数网格点 × 5,000 个伪实验 × 20 个多起始点 = **500 万次独立拟合**任务。设计任务调度方案，在 1,000+ CPU 核心的 HPC 集群上实现高效并行执行。
- **多起始点全局优化策略**：针对似然面存在多重局部极小值的问题，实现 20 个不同初始猜测值的并行扫描策略，对每次拟合取全局最优解，确保不遗漏真正的最优解。
- **经验分布构造与临界值提取**：对每个参数假设点，从 5,000 个伪实验的拟合结果中构造 Δχ² 的经验概率密度函数，通过数值积分提取满足给定置信水平（68.27%）的临界值，替代渐近近似的固定阈值。
- **覆盖率验证与参数区间敏感性分析**：系统研究了扫描区间选择对 FC 临界值的影响，发现窄扫描区间会遗漏替代全局极小值导致欠覆盖，据此确定了安全的扫描范围。

**Result**：
- 成功完成 500 万次拟合的全量 FC 构造。经验临界值为 **Δχ² ≈ 1.57**，显著高于渐近近似的 1.0，定量证实了传统方法在该场景下的失效。
- 关键参数的测量精度为 **1.4%**（50天数据），模型不确定性贡献 **< 0.3%**，证实统计误差占主导地位。
- 约 **8% 的伪实验**产生了不连续的置信区间，进一步揭示了渐近方法的局限性。
- 基于上述定量分析，向实验合作组提出了关于关键参数处理方式的统计策略建议，**该建议已被采纳**并体现在合作组的首篇物理结果文章中。整个统计推断方案具有完整的可复用性，可直接推广至其他大型实验的频率学置信区间构造。

## 项目四：JUNO 事例重建软件 OMILREC 的物理驱动加速

**项目角色**：核心开发者 / 性能优化负责人

**技术栈**：C++, Linux perf, Hotspot, Minuit, CMake

**项目描述**：

**Situation**：JUNO 中心探测器装备了约 1.8 万只大型 PMT 与 2.5 万只小型 PMT。事例重建软件 OMILREC 需要根据所有 PMT 的击中信息反推顶点位置与沉积能量——本质上是一个对全部 PMT 求和的极大似然拟合问题。随着实验长期运行所产生事例数将达到数十亿量级，OMILREC 的单事例耗时已成为离线数据处理流水线的关键瓶颈，对集群算力占用居高不下。

**Task**：在不修改物理算法逻辑、保证重建结果与原版本完全一致的前提下，对 OMILREC 进行系统性性能剖析与加速，目标是显著降低单事例处理耗时（≥ 2× 加速），为合作组的大规模重建任务节省可观的算力资源。

**Action**：
- **性能剖析定位真正瓶颈**：使用 Linux `perf` 与 Hotspot 工具在生产规模负载下采集函数级耗时分布与调用关系，借助火焰图直观呈现热点函数与重复计算路径，确保每一轮优化都基于量化数据而非经验直觉。
- **基于物理特性的稀疏化重建**：观察到对任一事例，只有靠近顶点的少量 PMT 真正接收到光子，远端 PMT 几乎没有贡献。据此引入"有效 PMT 筛选"机制——先依据事例的光强分布快速挑出真正有信号的 PMT，跳过对无贡献 PMT 的几何与时间计算，使单事例的计算量从"全探测器遍历"压缩到"仅相关区域"，是本项目最核心的物理直觉优化。
- **几何量的预计算与复用**：Minuit 似然循环中每次迭代都会重新计算 PMT 与顶点之间的几何距离、立体角等量。识别出这些量中相当一部分仅依赖探测器固有几何、与拟合参数无关，将其移出似然函数循环、在事例开始时一次性预计算，并在每次迭代中直接复用，消除了大量本可避免的重复运算。
- **几何运算的向量化与并行**：将 PMT 位置等几何信息按连续数组组织，使 CPU 能够一次对多只 PMT 同步执行加减、平方根等基本运算；同时在事例级别引入多核并行处理，进一步压缩单次重建耗时。
- **数值一致性回归测试**：搭建逐事例对比的回归测试，覆盖顶点位置、重建能量等所有物理输出量，确保所有优化在物理结果上完全无损。

**Result**：
- 经过多轮"剖析—优化—验证"迭代，OMILREC 单事例处理速度提升约 **3 倍**，在相同集群规模下离线重建流水线节省了大量算力资源，显著缓解了大规模数据处理的资源压力。
- 所有优化均通过逐事例数值一致性回归测试，重建物理量与原算法在浮点精度范围内完全一致（相对误差 ≲ 10⁻⁹），未引入任何物理偏差。
- 优化版本已被 JUNO 合作组采纳并应用于日常重建任务；相关方法论在 **第 26 届 JUNO 国际合作会议** 作主题报告（*Speedup of OMILREC*）。
