项目一：高性能能谱拟合框架的自研设计与性能优化
================================================================================

项目角色：框架独立设计者与核心开发者 / 张量化算法主创 / 性能优化负责人
技术栈：Python, PyTorch, NumPy, Linux Profiling, 张量化重构, 稀疏矩阵 (CSR), 分块计算 (Tiling), 函数级缓存 (LRU Cache), CPU 缓存局部性优化

【Situation】
课题面向反应堆中微子振荡参数的精确测量：需对实验观测能谱执行前向折叠（Forward Folding）拟合——即对"反应堆能谱 → IBD 微分截面 → 沉积能量谱 → 探测器响应 → 重建能谱"的多层卷积链路进行端到端建模，再与观测谱比对、提取振荡参数。该方法物理严谨，但计算开销极高：单次拟合涉及高维矩阵运算（560×5600 响应矩阵构建、5600×5600 矩阵乘法）与大量特殊函数（误差函数 erf）的密集调用，原始显式循环实现下耗时约 110 秒；而 Feldman-Cousins 频率学覆盖率构造需执行约 500 万次拟合（总规模约 2×10⁵ CPU·h），在常规算力下不可行，社区开源工具也无法直接覆盖该使用场景。

【Task】
从零自主设计并实现一套高性能拟合框架，将整条计算链路统一为可微分的张量算子流水线，在保证数值精度（相对误差 < 10⁻⁶）的前提下，使百万级任务在有限集群资源上可行。

【Action】
- 张量化算法设计（核心自研工作）：独立完成顶层架构设计，将原本"显式循环 + 标量函数调用"的物理链路重构为四级张量算子的级联流水线，使整条链路在 PyTorch 计算图上端到端可微分；将系统误差统一编码为冗余参数嵌入目标函数。
- 性能剖析：通过 Profiling 自顶向下分解单次拟合的时间分布，定位到信号谱计算占总耗时 98.6%（响应矩阵构造 66.7%、矩阵映射 30.8%），确定优化优先级。
- 分层缓存策略（约 3× 加速）：基于对计算图各张量"参数依赖性"的分析，将中间量分为静态量（拟合前一次性预计算并常驻内存）与动态量（基于 functools.lru_cache 函数级 memoization），消除参数扫描与 Toy-MC 中的重复计算。
- 带状稀疏矩阵（约 1.5× 加速）：分析并证实关键矩阵的非零元集中于主对角带，采用 PyTorch CSR 稀疏存储，将矩阵-向量乘法复杂度从 O(N²) 降至 O(N·w)（w 为带宽），显著降低内存占用与带宽压力。
- 分块张量化构造（约 1.5× 加速）：将 560×5600 响应矩阵沿列方向切分为固定宽度子块（基准测试确定最优块宽 512），逐块计算并累加，使中间临时张量从约 25 MB 降至 1 MB 级别、完整驻留 L2 缓存，最大化 CPU 缓存局部性；响应矩阵构造时间从约 63 ms 降至 27 ms。
- 数值正确性验证：在多档统计量下分别生成 5000 组 Toy-MC 验证收敛性，配合 Minuit2 的 is_valid / EDM / Hesse 正定性判据，最终成功率最高达 100%。

【Result】
- 单次拟合耗时由 110.1 s 降至 15.2 s，总体约 7× 加速（信号谱 8×、响应矩阵 9×、矩阵映射 6×），叠加张量化重构的初始加速后整体提升超过一个数量级。
- 双精度基准下相对误差控制在 10⁻⁶ 量级，Toy-MC 拟合收敛成功率达 100%。
- 框架成功支撑 500 万次拟合的全量任务，相关成果以第一作者身份发表于 EPJC，被国际合作组采纳为基础分析工具。

项目二：大规模数值模拟驱动的数据分箱策略优化
================================================================================

项目角色：核心开发者 / 数据策略研究负责人
技术栈：Python, NumPy, SciPy, 大规模 Monte Carlo 模拟, 协方差矩阵分析, 参数化降维

【Situation】
在大型实验的数据分析中，观测数据的分箱（binning）策略直接影响参数估计的偏差与精度：分箱过细导致边缘箱统计量稀疏、近似失效并引入系统性偏差；分箱过粗则丢失信息、降低灵敏度。模型分段策略需在"模型灵活性"与"外部数据约束能力"之间寻找最优平衡，本质是高维参数优化问题。

【Task】
通过大规模数值模拟，定量评估不同分箱与分段策略对关键参数测量精度的影响，给出可操作的最优策略并验证其稳健性。

【Action】
- Toy-MC 自动化仿真流水线：搭建支持 5000–10000 个伪实验的仿真平台，覆盖 56–560 bins 多种配置，实现参数扫描与结果统计的全自动化。
- 非参数化偏差评估：基于分位数方法（第 16/84 百分位）定义偏差指标，避免对分布正态性的假设依赖。
- 分段参数化与降维：将连续函数空间离散化为有限维段归一化因子，实现"段内刚性、段间自由"的参数化策略，将高维优化问题转化为可控的有限维参数估计。
- 对照实验设计：通过固定模型与扰动模型两组对照，系统量化模型依赖引入的额外系统误差。

【Result】
- 确定最优分箱宽度：调整 bin 宽后，关键参数的统计偏差由 0.8σ 降至 < 0.2σ，有效抑制低统计量下的近似失效问题。
- 确定模型最优分段宽度：在最优段宽附近，模型依赖系统误差降至 < 0.3%，实现模型灵活性与统计约束的最优折中。
- 研究成果被国际合作组采纳，直接应用于首篇正式发表论文的分析流程。

项目三：大规模 Feldman-Cousins 统计推断框架的工程化实现与集群部署
================================================================================

项目角色：核心开发者 / 统计推断方案负责人
技术栈：Python, PyTorch, HPC 集群调度, 分布式并行计算, 大规模 Monte Carlo, 似然比检验

【Situation】
精确参数测量中，传统渐近统计方法（Wilks 定理）在低统计量或目标函数高度非线性的场景下会失效——实际检验统计量分布显著偏离理论分布，导致置信区间覆盖率不正确，且目标函数存在多个局部极小值，单次点估计可能严重偏离真值。需采用 Feldman-Cousins（FC）方法通过海量伪实验构造经验分布，但计算规模极为庞大。

【Task】
设计并实现一套高并发的统计推断引擎，在 HPC 集群上完成百万级拟合任务，构造关键参数的经验分布并验证置信区间覆盖率。

【Action】
- 任务编排与并行调度：将 FC 构造分解为 50 参数网格点 × 5000 伪实验 × 20 起始点 = 500 万次独立拟合，在 1000+ CPU 核 HPC 集群上完成高效并行执行。
- 多起始点全局优化：针对目标函数多极小值问题，对每次拟合执行 20 个不同初值的并行扫描并取全局最优解，确保不遗漏真正最优解。
- 经验分布构造与临界值提取：对每个参数假设点，从 5000 个伪实验结果构造经验概率密度函数，通过数值积分提取置信水平 68.27% 的临界值，替代渐近近似阈值。
- 覆盖率敏感性分析：系统研究扫描区间选择对临界值的影响，定位"窄区间遗漏替代极小值导致欠覆盖"问题，确定安全扫描范围。

【Result】
- 成功完成 500 万次拟合的全量 FC 构造，经验临界值显著高于渐近近似（实测 1.57 vs. 理论 1.0），定量证实传统方法在该场景失效。
- 关键参数测量精度达 1.4%，模型不确定性贡献 < 0.3%，证实统计误差占主导。
- 约 8% 伪实验产生不连续置信区间，进一步揭示渐近方法的局限性。
- 基于定量分析向合作组提出统计策略建议，已被采纳并应用于首篇正式发表论文；整套方案具有完整可复用性，可推广至其他大规模置信区间构造场景。

项目四：大型 C++ 事例重建软件 OMILREC 的性能剖析与加速
================================================================================

项目角色：核心开发者 / 性能优化负责人
技术栈：C++, Linux perf, Hotspot, Minuit, CMake

【Situation】
OMILREC 是一个 C++ 实现的中等规模重建框架，需要根据约 4.3 万只传感器的击中信息逐事例反推顶点位置与能量（本质是对全部传感器求和的极大似然拟合）。长期运行将产生数十亿事例，单事例耗时已成为整条离线处理流水线的关键瓶颈，对集群算力占用居高不下。

【Task】
在不修改算法逻辑、保证结果与原版本完全一致的前提下，对 OMILREC 进行系统性性能剖析与加速，目标 ≥ 2× 加速，节省集群算力资源。

【Action】
- 性能剖析定位瓶颈：使用 Linux perf 与 Hotspot 在生产规模负载下采集函数级耗时分布与调用关系，借助火焰图直观呈现热点函数与重复计算路径，确保每轮优化都基于量化数据。
- 数据稀疏化：观察到对任一事例仅少量靠近顶点的传感器真正有信号、远端传感器贡献近似为零，引入"有效传感器筛选"机制——先依据事例信号分布快速过滤出有效集合，跳过对无贡献传感器的几何与时间计算，使单事例计算量从"全量遍历"压缩至"仅相关子集"。
- 循环不变量外提：似然循环每次迭代都会重新计算与拟合参数无关的几何量（传感器与顶点距离、立体角等），将其移出循环、事例开始时一次性预计算并在每次迭代中复用，消除大量重复运算。
- 向量化与多核并行：将几何信息按连续数组组织，使 CPU 能一次对多只传感器同步执行基本运算；在事例级别引入多核并行处理，进一步压缩单次重建耗时。
- 数值一致性回归测试：搭建逐事例对比的回归测试，覆盖所有重建输出量，确保所有优化在结果上完全无损。

【Result】
- 经过多轮"剖析—优化—验证"迭代，OMILREC 单事例处理速度提升约 3 倍，相同集群规模下离线重建流水线节省大量算力资源。
- 所有优化均通过逐事例回归测试，输出结果与原算法在浮点精度范围内完全一致（相对误差 ≲ 10⁻⁹）。
- 优化版本已被合作组采纳并应用于日常重建任务；相关方法论在第 26 届 JUNO 国际合作会议作主题报告（Speedup of OMILREC）。
